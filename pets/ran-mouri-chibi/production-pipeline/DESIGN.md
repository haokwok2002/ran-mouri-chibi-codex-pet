# 小兰 · 清晰 Q 版 — 第二版设计规范

## 版本隔离

第二版使用独立目录 `codex-v2`、独立 Pet ID `ran-chibi-clear-v2` 和独立安装目录。它不会读取、覆盖或替换第一版 `ran-watercolor`。

## 清晰度策略

- 比例改为约 3.5 头身，脸和标志性头发占据更多有效像素，但不做两头身婴儿化。
- 保留原图最关键的身份锚点：灰棕大眼、鹅蛋脸印象、暖棕蓬松卷发、顶部翘起发束、浅蓝外套、米白高领和藏蓝百褶裙。
- 使用中等粗细的深棕外轮廓、简化的内部线条和高对比色块，仅在色块内部保留克制水彩纹理。
- 每格角色限制在 `168×178` 以内，置于 `192×208` 单格中央，缩小一点并保留四周空间。
- 从 512 像素面板缩至宠物尺寸后进行轻量边缘锐化，主要改善眼睛、发束和外套轮廓。

## 动画原则

所有 74 个使用格都来自独立绘制帧。每组动作使用一张固定网格分镜板：

- `idle`：七帧眨眼循环。
- `running-right/left`：八相原地跑循环，左右版本镜像。
- `waving`：四帧小幅手腕挥动。
- `jumping`：五帧蹲下、起跳、腾空、下降、落地。
- `failed`：八帧克制的眉眼和肩部情绪变化。
- `waiting`：六帧逐级抬手、轻触发梢、放下。
- `running`：六帧左右手交替的工作动作。
- `review`：六帧视线与手指到下巴的思考动作。
- `look-*`：十六帧顺时针头部与视线方向。

整组动作只计算一次联合裁切和统一缩放。代码不会用单张图片平移、旋转或缩放生成动画；裙摆/骨盆人体锚点仅用于纠正模型在分镜格中的落点误差。

### 状态切换锚点

- 每个状态的首帧统一为身体中心 `x=96`、鞋底基线 `y=193`。
- `idle`、`waving`、`failed`、`waiting`、`running`、`review` 和两组 `look` 是站定动作，所有帧都保持同一鞋底基线。
- `running-right/left` 保留真实跑步腾空，`jumping` 保留真实跳跃高度，因此中间帧可以离开基线。
- `jumping` 第五帧是 Codex 会长时间显示的落地帧，强制回到 `y=193`，避免循环末尾悬空。

### 帧数与速度边界

Codex Pet V2 的状态帧数和逐帧时长由桌面客户端固定，`pet.json` 没有 FPS 或 duration 配置。`jumping` 固定读取五帧，这也是该状态的上限；当前时序为 `140/140/140/140/280 ms`。构建脚本生成的状态 GIF 使用相同的运行时时序，因此预览与客户端一致。两组 `look` 是由指针方向选择的静态方位格，GIF 只使用 `130 ms` 的质量检查扫帧速度。

`idle` 也采用客户端真实的六格时序：`1680/660/660/840/840/1920 ms`。为避免缓慢眯眼，完全闭眼只放在第二格的 `660 ms` 短槽，第三格立即恢复睁眼；后续格保留各自独立绘制的头发和服装微动，但眼睛保持睁开。可见眨眼过程由约 `3000 ms` 缩至 `660 ms`，约快 `4.5×`。

## 自动质量门槛

构建时自动执行：

- Codex Pet V2 图集结构、尺寸、透明空格、绿边检查。
- 每个状态的帧唯一性和相邻帧像素变化检查。
- 主体最大尺寸 `168×178` 检查。
- 实际宠物尺寸边缘清晰度检查，并与第一版做量化比较。
- 跨格孤立碎片的连通区域清理。
- 十一个状态首帧的身体中心和鞋底基线检查。
- 所有站定动作的逐帧鞋底漂移检查，以及跳跃落地帧检查。
